# 人物/话题故事线流程

用于用户要求总结某个群成员、某段八卦、某个话题或事件链路时。先用脚本抽取素材，再由 Agent 阅读素材做语义总结。

## 抽取命令

人物故事线：

```powershell
python .\scripts\wechat_analysis.py story --chat "<群聊名>" --person "<成员名>" --alias "<群昵称或外号>" --self-name "<自己的名称>"
```

话题故事线：

```powershell
python .\scripts\wechat_analysis.py story --chat "<群聊名>" --topic "<话题关键词>" --alias "<相关关键词>" --self-name "<自己的名称>"
```

可选日期范围：

```powershell
--start 2026-06-01 --end 2026-06-24
```

`--end` 包含当天全天；不传 `--start/--end` 时使用该会话当前解密库内全量范围。

## 分析规则

- 人物故事线优先按 sender、群昵称、备注、昵称和用户传入 `--alias` 匹配。
- 话题故事线按 topic、alias 和上下文窗口聚合，不要只看单句关键词。
- 先阅读 `story_*.md` 的高活跃日期、主题计数和相关上下文窗口，再总结。
- 总结要按“时间线 + 分支事件 + 关键人物关系 + 群友反应 + 结论”组织。
- 不要输出大段原始聊天记录；只引用极短短语或转述。
- 对私人关系、相亲、恋爱、家庭等内容，用“群聊中呈现/聊天中提到”表述，避免当成事实定论。

## 输出建议

- 先给一段 3-5 句总览。
- 再列 5-9 个故事节点，每个节点包含日期、标题、发生了什么、群友反应。
- 最后给人物画像或话题结论，用轻松但不攻击的语气。
- 如果要生图，读取 `story-handbook-prompt-template.md`。
